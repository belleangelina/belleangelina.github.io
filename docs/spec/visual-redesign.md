# 视觉重设计说明

## 1. 目标

本次重设计优先解决页面“功能可用但视觉粗糙”的问题。当前方向已经从早期的“文学杂志感”收敛为：**简洁但精美的个人写作站**。

目标不是继续增加功能，而是重做视觉气质、页面层级和阅读体验，让站点干净、轻盈、自然，同时保留足够的设计细节。

核心目标：

- 页面简洁，但不能像默认博客模板。
- 视觉清爽、轻盈、现代，有精致细节，但不厚重、不复古、不刻意文艺。
- 首页第一屏保持极简，避免左右分栏和信息堆叠。
- 列表页使用封面和标题形成作品入口感，而不是纯文字归档表。
- 阅读页专注正文，不显示摘要和日期。
- 移动端优先保证可读性和布局稳定。
- 保留现有 Astro 内容结构、路由、RSS、sitemap 和部署流程。

## 2. 风格共识

当前已确认的总方向：

```text
简洁但精美 / 清爽浅色 / 自然 / 轻盈 / 现代 / 有细节但不花哨
```

明确不采用：

- 不走精致文学杂志感。
- 不走私人手稿 / 书房感。
- 不走冷淡的纯阅读器风格。
- 不做厚重纸张质感。
- 不做过度文艺化文案。
- 不做复杂动效或炫技型视觉。

精美感主要来自：

- 排版层级
- 留白节奏
- 清爽配色
- 轻卡片层次
- 封面占位图系统
- 克制的 hover 和细线细节

后续讨论原则：只追问会影响整体设计树的关键问题。具体颜色、圆角、间距、阴影、hover、响应式细节等由实现时直接判断，不再逐项细问。

## 3. 色彩与暗色模式

色彩方向由实现把控，但必须符合以下原则。

浅色模式：

- 近白背景，不使用厚重米黄纸色。
- 使用极浅冷灰、灰蓝或淡紫灰作为层次。
- 强调色低饱和，偏淡紫灰 / 冷灰蓝体系。
- 页面整体应清爽，不偏复古。

暗色模式：

- 必须支持暗色模式。
- 不做简单反色。
- 背景可使用深蓝黑或深紫黑，而不是纯黑。
- 正文使用柔和灰白，避免高对比刺眼。
- 强调色延续浅色模式的淡紫灰 / 冷灰蓝气质。

## 4. 排版与形状

标题：

- 标题要有一点仪式感，但不夸张。
- 首页主标题可以比正文明显，但不要做成杂志封面式巨型标题。
- 阅读页标题层级清楚，但不显示摘要和日期。

整体形状：

- 介于圆润柔和和清冷利落之间。
- 按钮可以略圆，保持轻盈。
- 卡片使用中等圆角，不做软萌感。
- 线条要细、浅、干净。
- 阴影极轻，只用于层次，不做厚重浮起。

## 5. 顶部导航

顶部不显示站名，不显示 `Belle Angelina`，不显示 `BA`，不显示副标。

顶部只保留导航：

```text
首页 / 文章 / 长篇 / 短篇 / 记录 / 关于 / 主题切换
```

布局：

- 桌面端导航居中。
- 导航保持轻量、干净。
- 目前倾向继续使用轻微 sticky，但视觉上必须非常淡，不抢首页主标题和阅读正文。

## 6. 首页设计

首页第一屏确定为极简欢迎区。

已确认：

- 不做左右分栏。
- 不在右侧放最近更新列表。
- 不显示副标题。
- 不显示内容数量。
- 首页主标题先使用：`一些记录`。
- 第一屏只保留主标题和两个入口按钮。

推荐第一屏结构：

```text
一些记录

[进入目录] [读短篇]
```

按钮文字暂定：

```text
进入目录
读短篇
```

### 6.1 主要入口

主要入口不要做普通三张卡片，也不要做横向作品卡。

已确认方案：下滑后以普通滚动分段展示，每次视觉上主要看到一个入口。

```text
长篇
连续故事

短篇
独立片段

记录
日常整理
```

入口分段规则：

- 普通滚动分段，不使用 scroll snap。
- 不使用编号。
- 不使用英文小标签。
- 文案极短。
- 通过大留白、细线、标题层级形成设计感。

### 6.2 最近更新

首页保留最近更新，但必须极简。

已确认：

- 首页最近更新显示 2 条。
- 使用横向作品卡。
- 左侧为竖向封面或自动占位封面。
- 右侧显示标题和日期。
- 不显示摘要。
- 不放在首页第一屏右侧。

## 7. 文章列表与卡片

文章列表不再使用摘要型博客卡片，也不使用纯日期标题归档表。

已确认方案：**横向作品卡**。

卡片结构：

```text
[竖向封面]  标题
            日期 / 分类
```

规则：

- 左侧竖向封面。
- 右侧标题 + 日期 / 分类。
- 不显示摘要。
- 整张卡片可点击。
- 卡片是轻卡片：浅背景、细边框、轻 hover。
- 桌面可使用 2 列横向作品卡。
- 手机端 1 列，仍保持左封面右文字。

### 7.1 封面

内容模型已有 `cover` 字段，优先使用真实封面。

没有封面的文章必须使用自动生成占位封面，避免列表有的有图、有的没图导致视觉混乱。

占位封面方向：

- 极简淡渐变。
- 少量抽象线条、圆弧或光感细节。
- 不在封面上重复放大标题。
- 占位封面只承担视觉层次，不抢标题。

封面比例：

- 使用竖向封面。
- 约 3:4 或 4:5。
- 不使用横向 16:9 封面。

## 8. 阅读页

阅读页确定为精致正文页，但去掉摘要和日期。

阅读页结构：

```text
分类 / 返回入口
标题
细分隔线
正文
章节导航（仅长篇章节页）
```

规则：

- 阅读页不显示摘要。
- 阅读页不显示日期。
- 正文行宽保持舒适，不铺满屏幕。
- 标题与正文之间用留白和细线分隔。
- 引用、代码块、图片、列表样式保持清楚但克制。
- 长篇章节页保留上一章 / 返回目录 / 下一章导航。

摘要字段仍可保留在内容模型里，但当前视觉设计中：首页、列表页、阅读页都不展示摘要。

## 9. 页面结构总览

首页：

```text
一些记录
[进入目录] [读短篇]

长篇
连续故事

短篇
独立片段

记录
日常整理

最近更新
2 条横向作品卡
```

文章总览页：

```text
标题
分类入口或概览
横向作品卡列表
显示数量
不显示摘要
```

分类页：

```text
短篇 / 长篇 / 记录
数量
横向作品卡列表
不显示摘要
```

阅读页：

```text
分类入口
标题
正文
```

## 11. 验收标准

- 首页第一屏简洁，不像普通博客，也不堆信息。
- 首页没有右侧列表，没有站名 masthead，没有统计数字。
- 主要入口下滑逐段出现，有设计感但自然。
- 最近更新使用 2 条横向作品卡。
- 列表页使用横向作品卡，封面统一，有无真实封面都整齐。
- 全站不显示摘要。
- 阅读页不显示摘要和日期，正文专注舒适。
- 浅色模式清爽，暗色模式精致且可读。
- 不引入额外运行时依赖。
- 不破坏现有路由和内容发布规则。

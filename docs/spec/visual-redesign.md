# 视觉重设计说明

## 1. 目标

本次重设计优先解决页面“功能可用但视觉粗糙”的问题。目标不是继续增加功能，而是把站点推进到更像文学阅读站、个人写作杂志和长文本阅读空间的状态。

核心目标：

- 首页第一屏更有记忆点，不只是普通列表入口。
- 文章卡片更克制、更有层级，不使用过重边框和杂乱块状设计。
- 阅读页更正式，突出标题、元信息和正文节奏。
- 移动端优先保证可读性，正文行宽、字号、行距和间距稳定。
- 保留现有 Astro 内容结构、路由、RSS、sitemap 和部署流程。

## 2. 风格关键词

```text
文学杂志感 / 安静 / 大留白 / 纸张质感 / 暖色调 / 精致但不花哨
```

视觉方向不是炫技型门户，也不是传统博客模板；它应该像一个安静的阅读房间：标题有仪式感，列表像目录，正文像正式排版后的文学稿。

## 3. 色彩

浅色模式使用暖纸色背景、墨色正文和棕金色强调色。暗色模式使用低亮度深棕黑背景、柔和米色正文和低饱和金色强调色。

原则：

- 背景不要纯白，避免刺眼。
- 正文不要纯黑，避免过硬。
- 强调色只用于导航、链接、标签和细节线条。
- 卡片和阅读区域使用轻微透明度、柔和阴影和细边线。

## 4. 字体与排版

站点继续使用系统 serif 字体栈，优先保证 GitHub Pages 静态部署稳定，不引入外部字体依赖。

排版原则：

- 首页标题使用大字号、紧凑行高，形成文学杂志封面感。
- 正文使用较宽松行距，中文长文本保持舒展。
- 英文小标题、eyebrow、标签使用小字号和字距，作为杂志式辅助信息。
- 阅读页正文行宽控制在约 42rem，不追求铺满屏幕。

## 5. 页面结构

### 5.1 BaseLayout

- 顶部导航改为更像站点刊头的 masthead。
- 品牌区包含缩写标记、站名和一句短副标题。
- 导航保持现有信息架构：首页、文章、长篇、短篇、记录、关于。
- 保留浅色 / 暗色模式切换。
- Footer 增加简短站点说明和 RSS 入口。

### 5.2 首页

首页定位为入口页和文学站封面，不是完整文章列表。

推荐结构：

```text
大标题 / 简介 / 行动入口
右侧编辑手记式说明
分类入口卡片
最近更新目录
```

首页应展示内容数量，但数字只是辅助，不喧宾夺主。

### 5.3 文章列表页

文章总览和分类列表页应像目录页：清楚、轻、可扫读。

- 分类卡片作为入口。
- 文章卡片用整卡点击区域。
- 日期和分类信息弱化，标题优先。
- 摘要保留，但不要抢正文标题。

### 5.4 阅读页

阅读页应像正式文学稿页面：

- 顶部留出足够空间给分类、标题、日期、摘要。
- 标题和正文之间用细线或留白分隔。
- 正文段落间距加大，中文阅读不拥挤。
- 引用、代码块、图片、列表有清晰但克制的样式。
- 章节导航更像阅读器底部导航，而不是普通按钮。

## 6. 本次实现范围

优先修改：

- `src/styles/global.css`
- `src/layouts/BaseLayout.astro`
- `src/components/ArticleCard.astro`
- `src/pages/index.astro`
- `src/pages/articles/index.astro`
- `src/pages/articles/novels/index.astro`
- `src/pages/articles/shorts/index.astro`
- `src/pages/articles/notes/index.astro`
- 短篇、记录、长篇作品、卷、章节阅读页

不修改：

- 内容仓库结构
- 内容读取逻辑
- 文件命名规则
- `RainyGirl.md` 大小写
- `/articles/shorts/RainyGirl/` URL
- RSS / sitemap 生成逻辑
- GitHub Pages workflow 的核心部署方案

## 7. 验收标准

- 首页不像默认博客模板，第一屏有明显文学站气质。
- 列表页比原版更精致，但仍能快速进入阅读。
- 阅读页正文在手机和桌面上都舒适。
- 暗色模式仍可用，且不只是颜色反转。
- 不引入额外运行时依赖。
- 不破坏现有路由和内容发布规则。

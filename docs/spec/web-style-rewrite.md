# Web 风格重写规格

## 目标

本次重写以 `web` 目录的 `浮记 · Flux` 示例为视觉基准，适配 `belleangelina.github.io` 现有 Astro 内容站。

保留现有内容结构、静态路由、RSS、sitemap、GitHub Pages workflow 和独立文章详情页，不改成示例里的 hash SPA。

## 已确认方向

- 视觉参考 `web`，不是 `sample`。
- 保留 `web` 的核心气质：渐变标题、玻璃卡片、霓虹强调色、柔和发光按钮、滚动渐入。
- 暗色背景不要示例里的星点。
- 暗色背景不要明显极光光斑或高饱和彩色大背景。
- 暗色背景使用更干净、更高级、稍亮一点的深靛紫 / 蓝黑 / 冷灰纯 CSS 渐变。
- 支持浅色主题。
- 浅色主题使用浅雾紫、冷白、淡蓝粉，与暗色的粉 / 紫 / 蓝色系同源。
- 浅色主题不是简单反色，也不改成浅绿色。

## 页面策略

### 全局布局

- 顶部导航参考 `web` 的轻量 fixed nav。
- 保留站点实际路由：首页、文章、长篇、短篇、记录、关于。
- 保留主题切换按钮。
- 页脚参考 `web` 的居中短句和 meta 信息。

### 首页

首页改为 `web` 风格：

1. 居中 Hero
2. 渐变主标题
3. 简短副文案 / 打字机效果
4. 两个主要入口按钮
5. 最近内容卡片
6. 分类入口卡片

内容来自现有内容 API，不引入示例假数据。

### 文章列表

- 总览、长篇、短篇、记录统一使用玻璃卡片网格。
- 卡片展示分类、标题、摘要或日期信息。
- 玻璃效果使用轻透明和边框，不做过重模糊。

### 阅读页

- 保留独立详情页。
- 阅读页使用 `web` 的渐变标题、返回入口、正文排版和代码 / 引用样式。
- 长篇章节页保留上一章 / 返回目录 / 下一章导航。

## 性能约束

- 不新增运行时依赖。
- 不使用星点背景。
- 不使用 canvas、视频背景、rAF 背景动画。
- 打字机效果只在首页短文案使用，字符数量有限。
- 滚动渐入使用 `IntersectionObserver`，元素出现后取消观察。
- 动效只使用 `opacity`、`transform`、背景和边框变化。
- 尊重 `prefers-reduced-motion`。
- 构建验证时控制终端输出量。

# 站点优化审计

审计日期：2026-07-11。范围包括 Astro 源码、内容读取、公共布局、生成产物、GitHub Actions，以及桌面端和移动端运行状态。

## 已解决问题

| 方向 | 审计证据 | 处理结果 |
| --- | --- | --- |
| 构建可重复性 | 依赖全部使用 `latest`、没有锁文件，`npm ls` 报 5 个 `invalid` | 固定精确版本、提交 `package-lock.json`，Actions 改用 `npm ci` 和 npm cache |
| 类型安全 | Astro build 不执行完整类型检查；补跑后发现首页联合数组类型错误 | 增加 `astro check`，修复类型错误并纳入 `npm run verify` |
| 搜索与分享 | 页面缺少 canonical、`og:url`、文章发布时间和发现文件 | 补齐 canonical、Open Graph、Twitter、文章日期、robots、favicon 和 sitemap 声明 |
| 渐进增强 | 无 JavaScript 时 reveal 内容保持透明；首页副标题静态 HTML 为空 | reveal 默认可见，仅在 `.js` 下启用动画；保留可见和读屏静态文案 |
| 可访问性 | 浅色弱文本约 `3.20:1`；缺少统一焦点和 `aria-current` | 调整浅色文本/强调色、增加全局焦点、当前导航语义和减少动态效果处理 |
| 滚动性能 | 正文滚动时最多每帧同步写入一次 `localStorage`；导航样式每次 scroll 直接更新 | 阅读记录限制为每 300ms 一次，导航更新合并到 animation frame |
| 首屏体验 | 1365px 下首页标题出现单字孤行；缩短入场节奏后整体观感弱于原版 | 收紧标题尺寸与 hero 高度，保留无脚本静态内容，并恢复 1.1s 与 350/550/750ms 的原始入场节奏 |
| RSS 正确性 | 无日期内容每次构建都使用当前时间，形成伪更新 | 无有效日期时省略 `pubDate`，增加 Atom self link |
| 回归防护 | 只有人工构建，无产物级结构和链接检查 | 新增产物审计脚本与分支 CI，验证 canonical、内部链接、结构、RSS 和发现文件 |

## 有意未增加

- 全文搜索：当前公开内容规模很小，新增索引和前端脚本的成本高于收益；内容量明显增长后再评估。
- 访问分析：涉及隐私、数据保留和产品选择，没有明确需求时不引入第三方追踪。
- 默认封面或装饰图片：内容仓库当前没有 cover 资源，不使用无关占位图；已有 `cover` 字段会用于社交分享图。
- 内容 CMS 或数据库：Markdown 仓库和静态构建仍满足当前发布模型，不增加运行时服务。

## 验证入口

```bash
npm ci
npm run verify
```

完整验证应同时满足：类型检查无诊断、静态构建成功、产物审计通过、桌面和移动端关键流程通过、远端 `Validate site` workflow 成功。
